Inauguración del blog#

Hola, soy Enrique Pavón y esta es la primera entrada de lo que será mi blog a la hora de escribir este artículo. La idea es relatar mis experiencias, toma de decisiones, problemas y otros eventos dignos de relatar en los procesos de instalación, prueba o uso de software, conceptos y herramientas, entre otros.

La intención no es ser objetivo, pero sí riguroso y explicativo, para que mis experiencias puedan servir como referencia o componente de aprendizaje para otras personas, o que simplemente sea un entretenimiento para tener la mente activa y entretenida. El uso que el lector quiera darle a la lectura es su elección.

Téngase en cuenta que el enfoque es escribir el proceso de aprendizaje de uso de un determinado software, dónde la experiencia brilla por su ausencia. La tónica será no preparar los artículos, sino redactarlos conforme transcurren los acontecimientos sobre los que se escribe.

Con todo esto, bienvenidos seáis todos. Os agradezco el tiempo que dediqueis a leer este artículo.

Creando el blog#

Ya no me enredo más, vamos a instalar el blog. Para crear el blog me gustaría usar Hugo que es un generador de sitios estáticos y que tiene muy buena pinta.

Instalando Hugo#

Debemos tener Git instalado.

sudo apt install git

Para instalar Hugo, estando en Ubuntu como es mi caso, vamos a instalarlo a través de Snap.

sudo snap install hugo --channel=extended

El motivo de usar --channel=extended es el soporte para Sass.

Vamos a comprobar que se ha instalado bien:

hugo version

Creando el proyecto de Hugo#

Para crear el proyecto de Hugo, ejecutamos lo siguiente:

hugo new site <nombre_del_blog>
cd <nombre_del_blog>

Creamos un repositorio en GitHub, que será el que albergue todo el proyecto.

Ahora vamos a iniciar un repositorio de Git, y lo configuramos con el repositorio de GitHub.

git init # Iniciamos el repositorio
git add . # Añadimos todos los archivos
git commit -m "init" # Hacemos un commit
git branch -M main # Cambiamos a la rama main y la hacemos principal
git remote add origin git@github.com:<usuario>/<repositorio> # Añadimos el remoto
git push -u origin main # Enviamos los cambios a GitHub indicando que haga una rama remota

Y ya tendríamos nuestro repositorio configurado.

Añadiendo un tema#

Ahora que tenemos un blog creado, vamos a añadirle un tema. Yo voy a elegir este, pero hay muchos más.

Añadimos el tema a nuestro proyecto:

git submodule add --depth=1 https://github.com/adityatelange/hugo-PaperMod.git themes/PaperMod

Para ver el resultado de lo que llevamos hasta ahora:

hugo server -t PaperMod -w -D

Donde -t indica el tema a aplicar (después lo configuraremos para que sea permanente), -w indica que mire cambios en el proyecto y recree los archivos y -D indica que muestre también aquellos artículos marcados como borrador.

Para hacerlo permanente, añadimos al archivo config.toml la línea:

theme = 'PaperMod'

Añadiendo contenido#

Para crear un artículo nuevo, escribimos el siguiente comando:

hugo new posts/<nombre_del_articulo>.md

Lo editamos en content/posts/articulo.md. Nótese que se puede hacer una jerarquía de categorías con las carpetas, aunque meter todas las categorías dentro de la carpeta content/posts es una buena idea porque te deja el índice para páginas especiales, como puede ser una dedicada a tu portafolio o una página de contacto, así como pruebas de concepto y despliegues de proyectos por el estilo.

También recordar que Hugo utiliza Front Matter, lo que permite ajustar los metadatos de un documento en Markdown.

Por último, creamos el sitio estático con el comando:

hugo

Sí, así de simple, y nuestro sitio estará en la carpeta public/ disponible para que lo publiquemos en cualquier sitio.

Publicarlo en GitHub será menester de otro artículo.

Que vaya bien.
