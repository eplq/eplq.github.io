Instalando un terminal, intérprete y prompt nuevos

August 22, 2022

Acabo de instalar, otra vez, Ubuntu y me apetece cambiar cosas. Cambiar de emulador de terminal, de prompt y de intérprete de comandos concretamente.

La idea viene, no solo de tener sistema operativo fresco, sino también de un vídeo de Antonio Sarosi en el que hablaba de las tres cosas que vamos a instalar y configurar hoy.

Por ello, en esta entrada vamos a ver como podemos cambiar todo esto sin morir en el intento.

Emulador de terminal#

Y como comienzo, está bien cambiar de emulador de terminal. El electo es Alacritty.

Alacritty es un emulador de terminal, como pueden ser Gnome Terminal o XTerm, escrito en Rust y que tiene aceleración por GPU. Lo que me gusta realmente de Alacritty es que se puede configurar con un archivo YAML, lo que lo hace fácilmente configurable simplemente copiando un archivo.

Para instalar Alacritty, primero necesitamos Rust, que es el lenguaje en el que está programado:

curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

Seguimos las instrucciones del instalador y reiniciamos el terminal.

Ahora que tenemos Rust, utilizamos cargo para instalar Alacritty:

cargo install alacritty

Y ya lo tendríamos instalado. Ahora bien, cuando pulsamos Ctrl+Alt+T se sigue a abriendo Gnome Terminal, que no es el objetivo.

Para configurarlo, existe el comando gsettings. Este nos permite configurar todos los aspectos de Gnome.

En nuestro caso, los ajustes que nos interesan son:

org.gnome.desktop.default-applications.terminal exec
org.gnome.desktop.default-applications.terminal exec-arg

El primero define el binario del terminal por defecto que va a usar Gnome, y el segundo es el argumento que se va a añadir a cada ejecución cuando se llame para ejecutar un programa.

En este caso, el segundo tiene el valor -e, como podemos ver con el siguiente comando:

gsettings get org.gnome.desktop.default-applications.terminal exec-arg

Lo vamos a mantener pues, tal como podemos ver en los comandos gnome-terminal --help-terminal-options y alacritty --help, tienen la misma función.

Vamos a cambiar el terminal por defecto con el siguiente comando:

gsettings set org.gnome.desktop.default-applications.terminal exec $(which alacritty)

Y ahora si pulsamos Ctrl+Alt+T, veremos que nos sale Alacritty, en vez de Gnome Terminal.

Hemos tenido éxito con esta parte, vamos a configurar el terminal.

Configurando Alacritty#

Alacritty utiliza un archivo de configuración. Hay varias opciones, como bien pone en su página de GitHub. En este caso, utilizaremos $HOME/.config/alacritty/alacritty.yml.

Shell o intérprete de comandos#

Como intérprete de comandos, siempre me ha llamado la atención zsh porque provee varias mejoras sobre bash. Es el que vamos a usar.

Para instalarlo, utilizamos apt:

sudo apt install zsh

Ahora si ponemos zsh en el terminal, ya podremos usarlo, pero la gracia es que sea el intérprete por defecto. Hay que destacar que esto no nos impide ejecutar scripts en bash porque no lo hemos desinstalado.

Para ponerlo por defecto, existe el comando chsh. Las opciones válidas para poner como shell residen en el archivo /etc/shells.

Lo configuramos:

chsh -s /usr/bin/zsh $USER

Nos pide la contraseña, intro y ya lo tenemos. Para comprobarlo podemos verlo en el archivo /etc/passwd.

cat /etc/passwd | grep $USER

Sin embargo, cuando abrimos un terminal nuevo, nos sigue saliendo bash. Esto es porque tenemos que cambiar la configuración de Alacritty que, temporalmente, le pusimos bash.

shell:
	program: /usr/bin/zsh
	args:
		- --login

Y ya tendríamos nuestro intérprete instalado.

Ahora vamos a por la última parte, el prompt.

Instalando el prompt#

Voy a optar por Oh My Zsh porque es específico de zsh, lo que provee una compatibilidad absoluta. Además, tiene muchos, muchos temas.

sh -c "$(curl -fsSL https://raw.github.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"
